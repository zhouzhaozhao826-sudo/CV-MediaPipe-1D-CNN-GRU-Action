## 一、当前项目状态总结

### 1.1 项目核心

基于 MediaPipe Pose 的深蹲动作规范性评估系统，包含 **三条技术路线**：

- **规则法（Rule-Based）**：生物力学角度阈值 → 实时判断动作错误类型 → 用于实时摄像头场景
- **深度训练法（Deep Learning）**：1D-CNN + GRU → 端到端四分类 → 用于视频输入场景
- **融合法（Hybrid）**：规则法置信度 + 深度法概率加权融合 → 兼顾可解释性与准确率

最终目标：**规则法 vs 深度法 vs 融合法 → 三者对比实验 → 支撑论文**。

## 二、数据集设计方案深度对比

### 2.1 方案 A：长视频多动作 → 自动切片 → 标注 → 训练

**定义**：一个人连续做 N 次深蹲（如 50 次），录制成一个长视频。切片算法自动切割成 N 个独立动作段。

```
一个5分钟视频（50次深蹲）
        ↓ 切片算法（基于hip_height峰值检测）
  50个动作段 × 4个视频 = 200个训练样本
        ↓ 标注（规则法自动预标注 + 人工复核）
  训练集 .npz
```

**优点**：视频少（4个）、流程展示系统完整性
**缺点**：依赖切片质量、同视频内样本多样性低

### 2.2 方案 B：短视频单动作 → 直接生成样本 → 训练

**定义**：每类 50 个独立短视频，每个视频只包含**一次完整深蹲**。无需切片，目录名即标签。

```
50个短视频 × 4个类别 = 200个视频
        ↓ 无需切片（每个视频本身就是一次完整动作）
  200个视频 → 逐帧提取特征 → 线性插值到50帧
        ↓ 标签已知（目录名即类别）
  训练集 .npz（200个样本）
```

**优点**：不需要切片、不需要标注、数据多样性 > 方案A
**缺点**：录制工作量大、需要管理更多文件

### 2.3 选用建议

| 场景 | 推荐方案 |
|:---|:---|
| 快速跑通全流程 | 方案B（短视频，最快） |
| 展示系统完整性 | 方案A（长视频，展示切片+规则法能力） |
| 追求最高准确率 | 混合方案（方案B提供训练数据，方案A展示系统） |

## 三、三种方法对比：规则法 vs 深度法 vs 融合法

### 3.1 规则法（Rule-Based Method）

基于生物力学知识的显式判定系统。不依赖训练数据，直接对每帧姿态特征进行阈值判断。

```
摄像头/视频帧 → MediaPipe Pose → 33个3D关键点
                                      ↓
                              15维特征计算
                                      ↓
                            状态机判定当前阶段
                            （standing → descending → bottom → standing）
                                      ↓
                            规则判定（bottom帧）：
                              ├─ 膝角 > 95° → depth_insufficient
                              ├─ 膝间距 < 髋宽×0.85 → knee_valgus
                              └─ 躯干倾角 > 35° → torso_lean
                                      ↓
                              输出：错误标签 + 矫正建议 + 置信度
```

**代码位置**：`features.py`（特征）→ `repetition.py`（状态机）→ `rules.py`（规则判定）
**适用场景**：实时摄像头反馈、论文 baseline、自动预标注

### 3.2 深度训练法（Deep Learning Method）

数据驱动的端到端分类系统。从标注数据中自动学习时空特征模式。

```
训练阶段：
  标注样本 [N, 50, 15] → 1D-CNN（局部特征）→ GRU（时序建模）→ FC → 4类概率

推理阶段：
  新视频 → 切片/直接 → 50帧特征矩阵 → 加载模型 → 预测类别
```

**网络**：Conv1d(15→64)×2 → GRU(64→128) → Linear(128→64) → Linear(64→4)
**代码位置**：`model.py`（网络）→ `training.py`（训练/评估）
**适用场景**：离线视频分析、论文核心创新、高准确率需求

### 3.3 融合法（Hybrid Method）

规则法与深度法的加权融合，兼顾可解释性与准确率。

```
输入动作段
    ├─→ 规则法 → score_rule, label_rule, issues
    └─→ 深度法 → prob_dl, label_dl
                ↓
          融合决策（_fuse_prediction）：
          • 规则法置信度 > 85% → 采纳规则法
          • 深度法概率 > 90% → 采纳深度法
          • 两者一致 → 返回一致结果
          • 冲突 → α×rule + (1-α)×dl 加权
                ↓
          输出：最终标签 + 置信度 + 矫正建议
```

**代码位置**：`pipeline.py:454`（`_fuse_prediction`）
**适用场景**：论文最终系统、需要解释的分类场景

### 3.4 三者对比总结表

| 维度 | 规则法 | 深度训练法 | 融合法 |
|:---|:---|:---|:---|
| **原理** | 生物力学阈值 + 状态机 | 1D-CNN + GRU 端到端 | 规则 + DL 加权融合 |
| **是否需要训练** | 否 | 是（需标注数据） | 是（深度模型需训练） |
| **是否需要切片** | 否（逐帧判定） | 是（需要完整动作段） | 是 |
| **适用输入** | 实时摄像头流 | 离线视频文件 | 离线视频 + 需要解释的场景 |
| **准确率** | 中等（~65-75%） | 较高（~80-90%） | 最高（~85-92%） |
| **可解释性** | 强 | 弱（黑盒） | 强（规则法提供解释） |
| **代码入口** | `rules.py` | `model.py` + `training.py` | `pipeline.py:run_video_with_model` |

## 四、MediaPipe Pose 33关键点详解

### 4.1 33个关键点完整列表

MediaPipe Pose 基于 BlazePose 模型，输出 **33 个 3D 关键点**，覆盖全身主要关节和面部参考点。

```
索引  名称              索引  名称              索引  名称
 0    鼻尖 (nose)        11   左肩              22   (填充点)
 1    (填充点)           12   右肩              23   左髋 ★
 2    (填充点)           13   左肘 ★            24   右髋 ★
 3    (填充点)           14   右肘 ★            25   左膝 ★
 4    (填充点)           15   左腕              26   右膝 ★
 5    (填充点)           16   右腕              27   左踝 ★
 6    (填充点)           17   (填充点)          28   右踝 ★
 7    左耳               18   (填充点)          29   (填充点)
 8    右耳               19   (填充点)          30   (填充点)
 9    左嘴角             20   (填充点)          31   (填充点)
10    右嘴角             21   (填充点)          32   (填充点)
```

★ 标记的是深蹲评估中实际使用的关键点（12个）：
- **肩**: 11(左肩), 12(右肩)
- **肘**: 13(左肘), 14(右肘) — 辅助判断上肢姿态
- **腕**: 15(左腕), 16(右腕) — 辅助判断上肢姿态
- **髋**: 23(左髋), 24(右髋) — 核心，计算髋高度、膝角、髋角
- **膝**: 25(左膝), 26(右膝) — 核心，计算膝角、膝间距
- **踝**: 27(左踝), 28(右踝) — 核心，计算膝角、踝间距、躯干倾角

### 4.2 坐标系统

MediaPipe Pose 输出的是 **归一化 3D 坐标**：

| 轴 | 范围 | 含义 |
|:---|:---|:---|
| x | 0.0 ~ 1.0 | 水平位置，0=图像最左，1=图像最右 |
| y | 0.0 ~ 1.0 | **垂直位置，0=图像最上，1=图像最下** |
| z | 约 -0.5 ~ 0.5 | 深度，0=髋部中点平面，正值=更靠近镜头 |

> **关键注意**：y 轴方向与直觉相反——站立时髋部 y 值**小**（靠近画面顶部），下蹲时 y 值**大**（靠近画面底部）。所以规则法中"hip_height 增大 = 进入下蹲"。

### 4.3 深蹲评估中使用的12个关键点示意图

```
       11 ●──────────● 12     ← 肩 (计算肩宽、躯干倾角参考)
          │          │
       13 ●          ● 14     ← 肘 (计算肘角)
          │   TORSO  │
       15 ●          ● 16     ← 腕
          │          │
       23 ●──────────● 24     ← 髋 ★ 核心 (髋高度、膝角顶点、髋角顶点)
          │          │
          │   LEGS   │
       25 ●          ● 26     ← 膝 ★ 核心 (膝角、膝间距)
          │          │
       27 ●          ● 28     ← 踝 ★ 核心 (膝角终点、躯干倾角终点)
```

### 4.4 33关键点 → 15维特征的技术路线

MediaPipe 输出原始坐标 (x, y, z)，不能直接用于分类——它们受摄像头距离、人体比例、画面位置影响。需要转换为**尺度不变的几何量**。

#### 角度类特征（7个）—— 尺度不变

使用三点求夹角公式：`∠ABC = arccos( (BA·BC) / (|BA|×|BC|) )`

| 特征 | 关键点 A | 关键点 B（顶点） | 关键点 C | 含义 |
|:---|:---|:---|:---|:---|
| `left_knee_angle` | 左髋(23) | **左膝(25)** | 左踝(27) | 左膝弯曲角 |
| `right_knee_angle` | 右髋(24) | **右膝(26)** | 右踝(28) | 右膝弯曲角 |
| `left_hip_angle` | 左肩(11) | **左髋(23)** | 左膝(25) | 左髋屈曲角 |
| `right_hip_angle` | 右肩(12) | **右髋(24)** | 右膝(26) | 右髋屈曲角 |
| `left_elbow_angle` | 左肩(11) | **左肘(13)** | 左腕(15) | 左肘弯曲角 |
| `right_elbow_angle` | 右肩(12) | **右肘(14)** | 右腕(16) | 右肘弯曲角 |
| `torso_tilt_angle` | 肩中点 | **髋中点** | 踝中点 | 躯干前倾角 |

> 角度是尺度不变的——不管人离镜头远近，90° 的膝角始终是 90°。这是规则法能用固定阈值（膝角 > 95°、躯干 > 35°）判定所有人的数学基础。

#### 高度类特征（3个）—— 用于状态机

| 特征 | 计算方式 | 用途 |
|:---|:---|:---|
| `hip_height` | 左髋y + 右髋y / 2 | **状态机核心信号**，追踪下蹲深度 |
| `shoulder_height` | 左肩y + 右肩y / 2 | 辅助参考 |
| `ankle_height` | 左踝y + 右踝y / 2 | 辅助参考 |

> 注意 y 轴方向：站立时 `hip_height` 值小（髋在画面上方），下蹲时值大（髋移到画面下方）。

#### 距离类特征（3个）—— 用于归一化基准

| 特征 | 计算方式 | 用途 |
|:---|:---|:---|
| `shoulder_width` | ‖左肩 - 右肩‖ | **归一化分母**，消除体型差异 |
| `hip_width` | ‖左髋 - 右髋‖ | 膝内扣判定的参考基线 |
| `torso_length` | ‖肩中点 - 髋中点‖ | 辅助参考 |

#### 比值类特征（2个）—— 归一化后的关键指标

| 特征 | 计算方式 | 用途 |
|:---|:---|:---|
| `knee_gap_ratio` | \|左膝x - 右膝x\| / 肩宽 | **膝内扣核心指标** |
| `ankle_gap_ratio` | \|左踝x - 右踝x\| / 肩宽 | 辅助参考 |

> 用肩宽归一化后，不同体型的人可以直接用同一阈值比较。比如 `knee_gap_ratio < 0.85 × hip_width / shoulder_width` 表示膝关节内收超过正常范围。

#### 为什么这15维是完备的？

```
33个关键点 (x,y,z) = 99个原始数值
        ↓ 几何计算 + 领域知识
15维特征 = 角度(7) + 高度(3) + 距离(3) + 比值(2)
        ↓
兼顾: 尺度不变性 + 可解释性 + 低维度
```

这15维覆盖了深蹲评估所需的全部信息：下肢弯曲程度（膝角）、髋部运动幅度（髋高度、髋角）、躯干姿态（前倾角）、膝关节排列（膝间距比），同时通过比值归一化消除了体型和距离差异。

## 五、切片（Segmentation）的作用详解

### 5.1 为什么需要切片？

深度模型需要输入**固定长度的动作序列**（50帧），但原始视频是连续的帧流，包含多次深蹲和深蹲之间的站立间隙。切片的作用是**从连续帧流中找出每次完整深蹲的起止位置**。

```
原始视频帧流 (几千帧):
██░░██░░░░██░░██░░░░██░░████░░░░██░░██░░░░██░░██░░░░
  squat1    squat2    squat3    squat4    squat5

切片后:
[squat1: 60帧] [squat2: 55帧] [squat3: 58帧] ... → 统一重采样到50帧 → 训练样本
```

### 5.2 切片算法流程

`segment.py` 中的 `SquatSegmenter` 使用 **峰值检测 + 质量过滤** 策略：

#### 第1步：信号提取与平滑

```python
hip_series = [frame.hip_height for frame in feature_rows]  # 逐帧髋高度
smoothed = Savitzky-Golay滤波器(hip_series, 窗口=11, 阶数=2)
```

SG 滤波器在保留峰值形状的同时去除高频噪声，比简单移动平均更适合保留动作的起止边界。

#### 第2步：双峰检测

```python
# 检测髋高度峰值（y值越大=蹲得越深=底部）
bottom_indices = find_peaks(smoothed, distance=12帧, prominence=0.015)

# 检测髋高度谷值（y值越小=站得越直=顶部）
top_indices = find_peaks(-smoothed, distance=12帧)
```

#### 第3步：配对成型

每个底部峰 → 向前找最近的站立峰作为起点 → 向后找最近的站立峰作为终点：

```
         底部峰
           │
    ┌──────●──────┐
    │    /  \     │
    │   /    \    │
    ●──      ────●
  起点          终点
 (站立峰)      (站立峰)
```

#### 第4步：质量过滤（8条规则）

| 过滤规则 | 阈值 | 过滤掉的典型情况 |
|:---|:---|:---|
| 起止帧跨度 < 12帧 | min_distance=12 | 噪声导致的假峰值 |
| 动作时长范围 | 700ms ~ 3500ms | 过快（非深蹲）或过慢（停顿） |
| 髋部幅度 | ≥ 0.025（归一化值） | 微蹲、半蹲等不够深的动作 |
| 起止髋高差 | ≤ 0.08 | 开始和结束时身高不一致（姿态不稳） |
| 起始膝角 | ≥ 110° | 起始时膝盖弯曲（不是站立状态） |
| 结束膝角 | ≥ 110° | 结束时膝盖弯曲（未完全站起） |
| 底部膝角 | ≤ 170° | 到底部时膝盖没有弯曲（假蹲） |
| 膝角变化量 | ≥ 8° | 全程膝盖几乎没弯（非深蹲） |

#### 第5步：重叠处理

如果两个检测到的动作段在时间上重叠（共享帧），只保留第一个，第二个标记为 "overlap_with_accepted_segment" 并丢弃。

### 5.3 切片在三条路线中的角色

| 路线 | 是否需要切片 | 原因 |
|:---|:---|:---|
| **规则法 - 摄像头** | 不需要 | 逐帧判定，实时反馈，不需要完整动作段 |
| **规则法 - 视频** | 需要（用于分析） | 切片后可以对每次深蹲单独评估，生成逐次报告 |
| **深度训练法** | 必须 | 模型输入是 (50帧, 15维) 的固定长度矩阵，必须从视频中切出完整动作段 |
| **融合法** | 必须 | 同时需要规则法判定和深度法推理，都需要完整动作段 |

### 5.4 切片质量对下游的影响

```
切片质量 → 训练样本质量 → 模型准确率

切片太短: 只包含半个深蹲 → 模型学到错误模式
切片太长: 包含站立间隙 → 稀释了关键的运动信息
切片错位: bottom帧定位不准 → 规则法判定的帧不是真正的底部
```

这就是为什么 `SquatSegmenter` 有8条质量过滤规则——宁可丢掉一些边界样本，也不能让低质量切片进入训练集。

```python
def _fuse_prediction(rule_label, rule_score, dl_label, dl_probs, alpha=0.4):
    rule_conf = rule_score / 100.0           # 规则法置信度 [0,1]
    dl_conf = dl_probs.get(dl_label, 0.0)    # 深度法置信度 [0,1]
    
    if rule_conf > 0.85:      return rule_label, rule_conf    # 规则法极度自信
    if dl_conf > 0.9:         return dl_label, dl_conf        # 深度法极度自信
    if rule_label == dl_label: return rule_label, max(...)     # 两者一致
    
    # 不一致：加权融合所有可能标签
    for each label:
        fused = alpha * (1 if label==rule_label else 0) + (1-alpha) * dl_probs[label]
    return best_label, best_score
```

### 7.9 训练优化机制

`training.py` 中的 `train_model()` 集成了四项优化措施，在 `run_workflow.py` 中均可配置：

#### Early Stopping（早停）

监控验证集 loss。若连续 N 轮未下降，自动终止训练：

```python
TRAIN_EARLY_STOPPING_PATIENCE = 20  # 20轮不降则早停
```

训练日志中实时显示 patience 计数器：`patience=3/20`。

#### Learning Rate Scheduler（学习率衰减）

使用 `ReduceLROnPlateau`：验证 loss 连续 `lr_patience` 轮不降时，学习率乘以 `lr_factor`：

```python
TRAIN_LR_PATIENCE = 10   # 10轮不降则衰减
TRAIN_LR_FACTOR = 0.5    # 衰减为原来的50%
```

#### Weight Decay（权重衰减 / L2 正则化）

在 Adam 优化器中添加 L2 惩罚项，约束权重大小以缓解过拟合：

```python
TRAIN_WEIGHT_DECAY = 1e-4
```

#### Class Weights（类别权重）

自动统计训练集中各类样本数量，按反比例加权，防止模型偏向多数类：

```
weight_i = total_samples / (num_classes × count_i)
```

配置开关：

```python
TRAIN_USE_CLASS_WEIGHTS = True
```

训练启动时会打印实际权重值，如：`类别权重: {0: 0.8, 1: 1.2, 2: 1.3, 3: 0.9}`。
